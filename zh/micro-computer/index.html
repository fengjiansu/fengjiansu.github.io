上机

Dfh 不对 0dfh 可以
宏汇编定义在inc文件中

子程序和lib库

子程序名即过程名

  • 主程序中

且子程序要写在主程序 invoke ExitProcess,0之后
main ENDP 之前此时子程序只以子程序名 ENDP结尾

  • 子程序中

创建lib库

  • 创建新项目 静态库
  • 生成自定义文件masm
  • 在源文件中添加子程序文件生成解决方案
  • 找到生成的lib文件复制到要调用库文件的函数中去

编译cpp时出错 项目属性 链接器高级入口点处的值删除(之前设为main)

程序模块

子程序模块

子程序单独编写，汇编形成目标模块OBJ文件
连接时输入子程序模块文件名
用共用伪指令PUBLIC和外部伪指令EXTERN声明
PUBLIC 标识符 [,标识符 …]
;定义标识符的模块使用
EXTERN 标识符:类型 [,标识符:类型 …]
;调用标识符的模块使用

子程序库

统一管理子程序模块
使用子程序库：在连接主程序模块时提供子程序库文件名
子程序文件编写完成、汇编形成目标模块
VS201X环境中编译生成.obj文件
利用库管理工具程序LIB.EXE：把子程序模块逐个加入到子程序库（.LIB）

库文件包含

直接在主程序源文件中用库文件包含伪指令INCLUDELIB说明
不用在连接时输入库文件名

宏汇编

宏（Macro）：具有宏名的一段汇编语句序列
宏需要先定义
宏名 MACRO 形参表
…… ;宏定义体
ENDM
然后程序中进行宏调用
宏名 实体参数
在汇编时，宏指令被汇编程序用宏定义的代码序列替代，实现宏展开

源文件包含

将INCLUDE伪指令指定的文本文件内容插入源程序
可以包含任何文本文件
一些常用的或有价值的宏定义存放在.MAC宏定义文件
各种常量定义、声明语句等组织在.INC包含文件
常用的子程序形成.ASM汇编语言源文件
利用INCLUDE伪指令包含其他文件，其实质仍然是一个源程序，只不过是分在了几个文件书写

模块连接

各种语言的程序分别编写
利用各自的开发环境编译形成OBJ模块文件
将它们连接在一起，生成可执行文件
为了保证正确连接
采用一致的调用规范 如c
声明共用函数和变量
正确传递入口参数和返回参数
使用MASM的扩展过程定义

cpp 里 extern “C” { long mean(long d[], long num); }

asm文件
mean proto d:ptr dword,num:dword ; 过程声明

mean proc USES ebx ecx edx,d:ptr dword,num:dword ; 过程定义

点击编译生成obj文件 在工程目录下的debug文件夹下找到文件
放在和cpp文件同级目录下
注意两个文件命名不能相同否则obj文件会混合在一起编译失败

调用API函数

MessageBoxA PROTO :DWORD,:DWORD,:DWORD,:DWORD
MessageBox equ
NULL equ 0
MB_YESNOCANCEL equ 3
MB_OK equ 0
MB_ICONWARNING equ 48
参数相加 例如 欧取消为3 感叹号是48则 48+3=51为感叹号加取消按钮
